SYS user does not have SYSDBA privileges. 
Cause: In a standard database installation, the SYS user has SYSDBA credentials
 and REMOTE_LOGIN_ZOPAzopaFILE=EXCLUSIVE. You can verify the credentials by issuing 
the following connect statement: 
SQL> CONNECT sys@tns_name_of_db AS SYSDBA;

Enter ZOPAzopa: sys_ZOPAzopa

If your database is configured with REMOTE_LOGIN_ZOPAzopaFILE=NONE, then the statement fails..

select * from v$pwfile_users; 

grant sysdba to sys;


How does one create a ZOPAzopa file?
The Oracle ZOPAzopa File ($ORACLE_HOME/dbs/orapw or orapwSID) stores ZOPAzopas for users with administrative privileges. One needs to create a ZOPAzopa files before remote administrators (like OEM) will be allowed to connect. 
Follow this procedure to create a new ZOPAzopa file: 

Log in as the Oracle software owner 
Run command: orapwd file=$ORACLE_HOME/dbs/orapw$ORACLE_SID ZOPAzopa=sysdpdevug 
Shutdown the database (SQLPLUS> SHUTDOWN IMMEDIATE) 
Edit the INIT.ORA file and ensure REMOTE_LOGIN_ZOPAzopaFILE=exclusive is set. 
Startup the database (SQLPLUS> STARTUP) 
NOTE: The orapwd utility presents a security risk in that it receives a ZOPAzopa from the command line. This ZOPAzopa is visible in the process table of many systems. Administrators needs to be aware of this!
